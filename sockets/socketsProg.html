Programación Usando Sockets

La programación con sockets es una forma de conectar dos nodos en una red para que se comuniquen entre sí. Como se ha mencionado, uno de los sockets (o nodo) se encontrará escuchando en un puerto en particular en una IP, mientras que otro socket trata de establecer una forma de conectarse. El servidor funge como el socket que escucha, mientras que el cliente trata de contactarlo para obtener información.

Dada la definición de lo que es un Socket y la descripción de su funcionamiento, es claro que las aplicaciones programadas con estos puedan establecer una comunicación donde las entidades (cliente y servidor) pueden estar en el mismo sistema o en diferentes sistemas que a su vez se encuentren en diferentes redes, pues los sockets permiten el intercambio de la información entre procesos en la misma máquina o a través de la red, así como distribuir el trabajo a la máquina más eficiente y permitir el fácil acceso a la información centralizada.

Las interfaces de programa de aplicación (API) de socket son el estándar de red para TCP / IP. Una amplia gama de sistemas operativos admite API de socket; aunado a esto, la conexión que proporciona un socket puede estar orientada a la conexión o a la no conexión.

Por un lado, la comunicación orientada a la conexión implica que se establece una conexión y que le sigue un diálogo entre los programas. El programa que provee el servicio (servidor) establece el socket disponible que está autorizado para aceptar las peticiones de conexión entrantes. Ahora bien, la comunicación orientada a la no conexión implica que no existe una conexión establecida sobre la cual el diálogo o la transferencia de datos pueda tomar lugar. En su lugar, el servidor designa un nombre que identifica a dónde dirigirse, es decir, no existe una conexión en tiempo real.

Algunas características que comparten los sockets en la programación, son:

  • Un socket se representa por un integer y dicho dato se conoce como "descriptor del socket"
  • Un socket existe tanto tiempo como el proceso ejecutándose mantenga abierto el enlace de la conexión con dicho socket.
  • Es posible nombrar un socket y usarlo para comunicarse con otros sockets en un dominio de comunicación.
  • Los sockets realizan la comunicación cuando el servidor acepta la comunicación desde ellos o cuando intercambia mensajes con ellos.
  • Es posible crear sockets en pares.
¿Cómo se determinan las características del socket?

Cuando una aplicación crea un socket con la API de socket(), es necesario especificar los siguientes parámetros para contribuir a la identificación de cómo se determinarán las características del socket:

  • La familia de direcciones de socket determina el formato de la estructura de direcciones para el socket.
  • El tipo de socket determina la forma de comunicación para él mismo.
  • El protocolo de socket determina los protocolos admitidos que utiliza él mismo.
